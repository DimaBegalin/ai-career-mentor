# WhatsApp-бот доставки чек-листа (WABA + n8n)

## Как работает вся цепочка

```
Ученик оставил WhatsApp-номер на сайте
        │  POST (номер + готовый текст чек-листа)
        ▼
n8n: Webhook /career-lead → лид сохранён → бот шлёт ШАБЛОН с кнопкой
        │
        ▼
Ученику в WhatsApp приходит: «Привет, {имя}! Твой чек-лист по {вузы} готов»
и кнопка «Получить чек-лист»
        │  ученик нажимает кнопку (открывается 24-часовое окно диалога)
        ▼
n8n: Webhook /wa-inbound → бот отправляет полный чек-лист текстом
        │
        ▼
Ученик отвечает боту → менеджер записывает на детальную профориентацию
```

Почему именно так: по правилам WhatsApp Business API бизнес **не может** первым
отправить произвольный текст — первое сообщение обязано быть одобренным
**шаблоном**. Зато после ответа/нажатия кнопки открывается 24-часовое окно,
в котором можно слать что угодно — туда и уходит полный чек-лист.

---

## Шаг 1. Получить доступ к WhatsApp Cloud API (WABA)

1. Зайдите на **developers.facebook.com** → My Apps → Create App → тип **Business**.
2. В приложении добавьте продукт **WhatsApp**.
3. На странице *WhatsApp → API Setup* возьмите:
   - **Phone number ID** (это не сам номер, а ID) → понадобится в n8n;
   - временный токен для теста. Для продакшена создайте **постоянный токен**:
     Business Settings → System Users → создать → выдать права `whatsapp_business_messaging` → Generate Token.
4. Привяжите свой реальный номер (или используйте тестовый номер Meta —
   он умеет слать только на 5 заранее добавленных номеров, удобно для отладки).

## Шаг 2. Зарегистрировать шаблон сообщения

WhatsApp Manager → Message Templates → Create:

- **Имя:** `career_checklist_ready` (именно так — оно зашито в workflow)
- **Категория:** Utility
- **Язык:** Russian
- **Body:**
  ```
  Привет, {{1}}! 🎓 Твой персональный чек-лист поступления готов.
  Топ вузов под твой профиль: {{2}}.
  Нажми кнопку — пришлю чек-лист прямо в этот чат.
  ```
- **Кнопка:** Quick Reply → текст `Получить чек-лист`

Отправьте на модерацию (обычно одобряют за минуты–часы).
Важно: переменные {{1}} и {{2}} не могут содержать переносы строк —
сайт уже передаёт их одной строкой.

## Шаг 3. Импортировать workflow в n8n

1. В n8n: **Workflows → Import from File** → выберите `n8n-whatsapp-checklist-bot.json`.
2. В двух HTTP-нодах (`WA · Отправить шаблон с кнопкой` и `WA · Отправить чек-лист`) замените:
   - `PASTE_PHONE_NUMBER_ID` в URL → ваш Phone number ID;
   - `PASTE_WABA_TOKEN` в заголовке Authorization → ваш постоянный токен.
3. **Активируйте** workflow (переключатель Active). Это важно дважды:
   - production-URL вебхуков (`/webhook/...`) работает только у активного воркфлоу;
   - лиды хранятся в staticData, которая живёт только в активном воркфлоу.

Ваши боевые URL после активации:
- приём лидов: `https://dimkame.app.n8n.cloud/webhook/career-lead`
- входящие из WhatsApp: `https://dimkame.app.n8n.cloud/webhook/wa-inbound`

(В режиме теста n8n использует `/webhook-test/...` — не перепутайте.)

## Шаг 4. Подключить webhook в Meta

Приложение Meta → WhatsApp → Configuration → Webhooks:
- **Callback URL:** `https://dimkame.app.n8n.cloud/webhook/wa-inbound`
- **Verify token:** `career-mentor-verify` (зашит в ноде «Ответить hub.challenge»;
  можно поменять — тогда поменяйте в обоих местах)
- Нажмите Verify and Save → подпишитесь на поле **messages**.

## Шаг 5. Сайт

В `index.html` эндпоинт уже прописан:
```js
const LEAD_ENDPOINT = "https://dimkame.app.n8n.cloud/webhook/career-lead";
```
Сайт сам нормализует номер в формат wa_id (8 7XX… → 7 7XX…, без «+»)
и передаёт **готовый текст чек-листа** — n8n ничего не генерирует,
только сохраняет и отправляет.

## Тест всей цепочки

1. Добавьте свой номер в тестовые получатели (API Setup → To).
2. Пройдите квест на сайте, оставьте этот номер.
3. В WhatsApp придёт шаблон с кнопкой → нажмите → придёт чек-лист.
4. В n8n → Executions видно оба срабатывания и возможные ошибки.

## Что доработать для продакшена

- **Хранилище лидов.** Сейчас лиды живут в staticData воркфлоу — при
  деактивации/переносе они обнулятся. Добавьте после ноды «Сохранить лид»
  ноду Google Sheets (Append Row) — получите заодно мини-CRM, а в ноде
  «Разобрать входящее» замените чтение staticData на Lookup по таблице.
- **Уведомление менеджера.** Параллельно с отправкой шаблона добавьте ноду
  Telegram/Slack: «Новый лид: имя, номер, топ-вузы» — менеджер перезванивает
  для записи на профориентацию.
- **Чек-лист как PDF.** Вместо текста можно слать документ: type: "document"
  с link на сгенерированный PDF — красивее для пересылки родителям.
- **Ответ «записаться».** Сейчас любое сообщение после кнопки получает чек-лист
  повторно. Добавьте IF-ноду: если lead.delivered === true — слать не чек-лист,
  а «Менеджер свяжется с тобой для записи на профориентацию» + уведомление менеджеру.
